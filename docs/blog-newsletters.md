# Blog newsletter submission list

Candidate curated newsletters for post submissions. **Do not submit until the blog has 3-4 niche-cluster posts.** On the publication day of each post, pick 3-5 entries whose scope matches the post's bucket.

Before first use, verify each entry still exists and still accepts submissions. Prune dead ones in place.

| Newsletter | Match | Submission |
|------------|-------|-----------|
| SRE Weekly | Buckets 1, 2 | https://sreweekly.com/submit/ |
| DevOps'ish | Buckets 2, 3 | https://devopsish.com (submission form at page bottom) |
| Last Week in AWS | Buckets 2 | https://lastweekinaws.com/newsletter/ |
| Python Weekly | Bucket 1 (Python-heavy) | email to the editor per https://www.pythonweekly.com/ |
| PyCoders Weekly | Bucket 1 (Python-heavy) | https://pycoders.com/submit-an-article |
| Console.dev | Buckets 1, 2, 4 (high bar) | https://console.dev/submit/ |
| Pointer.io | Buckets 2, 4 | https://www.pointer.io |
| DevOps Weekly (Rushgrove) | Buckets 2, 3 | weekly@devopsweekly.com |
| Hacker Newsletter | any (only if post lands on HN organically) | https://hackernewsletter.com |

## Rules

- Submission goes out on the **day of publication**, not later — editors run weekly cycles.
- Never submit a post to a newsletter whose scope it does not match. It burns goodwill and the editor remembers.
- Track submissions in a personal note if desired, but nothing repo-visible is required.
